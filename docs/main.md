# 项目文档：鳌太线模拟器

## 1. 项目愿景

- **主题：** 以秦岭“鳌太线”真实徒步路线为背景的生存策略游戏。
- **核心体验：** 资源博弈、极简策略、对自然的敬畏。
- **平台：** 轻量化 Web 网页版（适配 PC/手机浏览器）。
- **美术风格：** 2D 矢量艺术风格，结合侧面海拔剖面视图。

---

## 2. 核心玩法逻辑

游戏分为 **[筹备期]** 与 **[挑战期]** 两个阶段，并辅以 **[Roguelike 循环]**。

### A. 筹备期 (Preparation Phase)

- **人物选择：** 从 10+ 个具有不同预算、体质、技能的角色中选择主角。
- **组建团队：** 挑选或由系统随机分配队友（队友属性影响团队士气与突发事件走向）。
- **物资采购（背包管理）：**
- **核心机制：** 空间解谜（类似《生化危机》方格背包）。
- **约束条件：** 必须在**预算上限**、**空间容量**和**负重上限**（影响体力消耗速度）之间寻求平衡。
- **关键装备：** 帐篷、睡袋、炉头、燃料、脱水蔬菜、急救包、卫星电话等。

### B. 挑战期 (Expedition Phase)

- **视觉表现：** 采用侧面剖面视角。角色在代表海拔起伏的折线图上移动。
- **交互方式：** 节点式移动。玩家点击下一个目标点（营地/地标），系统根据路线斜率和天气结算过程。
- **核心数值管理：**
- **体温：** 受天气和海拔影响，低于临界值会导致失温（Game Over）。
- **体力：** 受负重和地形斜率影响，耗尽将无法移动。
- **血氧：** 随海拔升高衰减，影响决策成功率。

- **决策节点：** 每个节点面临：继续赶路、原地扎营（消耗物资）、紧急撤回。

### C. 结算与继承 (Meta-Progression)

- **评分系统：** 根据到达终点的时间、全员生还率、剩余物资计算。
- **遗产机制：** 高分可解锁更轻量化的“神级装备”或特殊角色背景，带入下一局游戏。

---

## 3. 视觉与技术方案

- **地图：** 使用基于真实地理数据的 2D 折线剖面图，标注关键地标（塘口、鳌山、石海、拔仙台等）。
- **UI 设计：** 极简主义。主色调建议为高对比度的黑/白/灰，搭配警示红（失温提示）和安全绿。
- **开发技术栈建议：** \* 前端：`React` 或 `Vue` 配合 `Canvas/PixiJS` 渲染地图。
- 逻辑：状态机驱动（State Machine）处理天气与随机事件。

## 4. 待办事项 (To-Do List)

### 阶段一：原型设计 (MVP)

- [ ] **数值建模：** 设定基础角色的体力、体温下降曲线公式。
- [ ] **关卡节点：** 整理 10-15 个真实地标的海拔数据及里程。
- [ ] **随机池：** 编写首批 20 个突发事件脚本（如：暴雪失联、队友扭伤、水源污染）。

### 阶段二：系统开发

- [ ] **开发背包系统：** 实现方格占位与负重计算逻辑。
- [ ] **开发地图组件：** 实现折线图渲染及角色图标沿线移动动画。
- [ ] **开发决策系统：** 实现不同天气/地形下的成功率判定。

### 阶段三：美术与内容

- [ ] **角色立绘：** 创作 10 个以上符合 2D 矢量风格的人物头像。
- [ ] **UI 界面：** 设计主菜单、背包界面、结算画面。

## 5. 项目后续思考 (Icebox)

- **道德困境：** 是否加入在极端情况下“抛弃队友”导致法律或心理惩罚的机制？
- **真实性引导：** 增加“户外知识百科”，向玩家普及鳌太线的法律禁令及安全意识。

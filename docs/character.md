在选择角色时，我们可以暂时只显示主要属性即可。按照你的建议。我感觉可以提供如下内容：
1，角色立绘（可以是随机选择一个AI绘制的人物即可），以及年龄和性别（和立绘一样对游戏没有直接影响，单纯用于增加代入感）
2，初始装备（后续可以卖出换成其他装备，但卖出会折价）
3，体力，影响每次移动的最大距离，组队时，会综合多个角色计算出最终体力
4，意志，减少天气对于属性的惩罚，组队时，会综合计算
5，领导，只在作为队长时有用，可以提升小队属性
6，户外技能，影响事件成功率
7，负重，字面意思。

除此之外，角色还有一些状态数值，用来和游戏中的其他环节交互：
1，健康，角色生病或受伤会降低健康，从而降低所有属性。
2，氧气，氧气影响体力恢复的速度
3，体温，体温影响体力上限
4，饮水和饱腹，这两项状态如果低于阈值，会导致体力无法自动恢复，即便在营地中。
5，精神压力，如果角色崩溃，会导致无法受玩家控制，可能会做出随机行为（例如冒险前进或直接返回等）

### 1. 核心属性（Primary Attributes）

- **关于“体力 (Stamina)”与“负重 (Carry Weight)”的联动：**
- 建议引入一个**“过载阈值”**。当负重超过角色的 80% 时，体力的消耗速率增加；超过 100% 时，移动距离减半。

- **关于“组队综合计算”的算法建议：**
- **体力：** 建议采用“木桶效应（最小值）”或“加权平均”。在徒步中，队伍的速度通常取决于最慢的那个人。
- **意志：** 建议采用“平均值”。一个意志坚定的队友可以鼓励他人，但一个崩溃的队友也会拖累全队的士气。

- **“领导 (Leadership)”的实战化：**
- 除了提升属性，领导力可以赋予玩家**“特殊指令”**。例如：在意志力临界点时使用“强制扎营”或“动员演说”，以此暂时锁定精神压力不上升。

### 2. 状态数值（State Values / Survival Engine）

这是游戏最出彩的“硬核”部分，我们可以进一步细化它们之间的**连锁反应（Combo Effects）**：

- **体温 体力上限（核心致命逻辑）：**
- 这是非常真实的设计。当体温降低（失温开始），体力条的上限可以变成**灰色**并不断缩短。即便休息，也只能回复到那个缩短后的上限，除非体温回升。

- **氧气 体力恢复速度：**
- 在剖面图中，海拔越高，氧气值越低。玩家会发现：在海拔 3500 米的地方，即便原地休息 1 小时，体力也只能恢复 10%，而在山脚下能恢复 50%。这强迫玩家必须精准计算每一滴体力的使用。

- **压力 (Stress) 失控行为：**
- 这个机制让游戏具有了恐怖游戏的色彩。

### 3. “疲劳值 (Fatigue)”

- **逻辑：** 每天的移动都会产生不可逆的疲劳。
- **影响：** 疲劳值越高，健康度下降越快，且更容易在随机事件中受伤（如：崴脚）。
- **消除：** 只有在特定营地进行“深度睡眠”（消耗大量食物和燃料）才能显著降低。

### 角色系统模型

| 类别         | 数值名称               | 核心机制映射                                     |
| ------------ | ---------------------- | ------------------------------------------------ |
| **生存约束** | 负重、金钱、装备       | 决定了“容错空间”，是进山前博弈的核心。           |
| **基础属性** | 体力、意志、领导、技能 | 决定了“小队基础性能”，影响移动效率和判定成功率。 |
| **实时生理** | 体温、氧气、饱腹、饮水 | 动态压力源，直接干预体力条的长度和恢复。         |
| **风险指标** | 健康、压力、疲劳       | 长期风险，触发减益状态或“失控”结局。             |
